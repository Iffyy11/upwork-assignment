Web Accessibility Data

This page demonstrates accessible data presentation using semantic HTML, proper tables, and lists. All content is keyboard navigable.

WCAG 2.1 Conformance Levels

The Web Content Accessibility Guidelines (WCAG) 2.1 defines three levels of conformance. This project targets Level AA compliance.

WCAG 2.1 Conformance Requirements
Level Description Requirements
A Basic accessibility Text alternatives, keyboard access
AA Ideal for most websites 4.5:1 contrast, captions, headings
AAA Highest accessibility 7:1 contrast, sign language

Color Contrast Requirements

WCAG AA Standards

Normal Text (under 18pt)
Minimum 4.5:1 contrast ratio
Large Text (18pt and larger)
Minimum 3:1 contrast ratio
UI Components
Minimum 3:1 contrast ratio

Keyboard Shortcuts

Navigation Keys

  • Tab - Move to next element
  • Shift + Tab - Move to previous element
  • Enter - Activate links and buttons
  • Space - Activate buttons, toggle checkboxes
  • Arrow Keys - Navigate within components
  • Esc - Close dialogs and menus

Screen Reader Keys (NVDA/JAWS)

  • H - Navigate by headings
  • L - Navigate by lists
  • T - Navigate by tables
  • F - Navigate by form fields

Accessibility Statistics

  • Accessible sites have 50% higher engagement
